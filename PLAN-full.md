## פיצ'ר 2: דיווח כמותי ביומן עבודה

### מה משתנה

**WorkLogForm.jsx — טופס מנהל עבודה:**

במקום בחירת קטגוריה → **בחירת משימה ספציפית** מרשימת משימות הפרויקט.

כל שורת דיווח:
```
משימה:    [dropdown — רשימת משימות הפרויקט]     ← חדש
כמות:     [15]  מתוך 120 מ"ק (נותרו: 45)        ← חדש
עובדים:   [6]
תיאור:    יציקת תקרה קומה א׳
```

המשימות מגיעות מ-`projectTasks` — עובד זהה עם מחירון ועם כת"כ.

**store.js — addWorkLog:**

כל entry ביומן כולל עכשיו:
```
entry = {
  taskId: 123,           ← חדש
  taskName: "בטון B30",  ← חדש
  category: "עבודות שלד",
  unit: "מ\"ק",          ← חדש
  executedQty: 15,        ← חדש
  workersCount: 6,
  description: "יציקת תקרה",
  clientCost: ...,
  actualCost: ...,
}
```

**חישוב אחוז ביצוע per משימה:**

פונקציה חדשה ב-store.js:
```
getTaskProgress(taskId) {
  כל היומנים → סינון entries עם taskId → סכום executedQty
  return { executed: 75, planned: 120, percent: 62.5 }
}
```

**WorkLogs.jsx — תצוגת יומנים:**

במקום badge קטגוריה → מראה: משימה + כמות שבוצעה
```
בטון B30 — 15 מ"ק | 6 עובדים | יציקת תקרה
```

**תאימות אחורה:**
- יומנים ישנים (בלי taskId) ממשיכים לעבוד כמו קודם
- יומנים חדשים כוללים taskId + executedQty

## פיצ'ר 3: כרטיס קטגוריה — תמונת מצב

### דף חדש או שדרוג ProjectTasks.jsx

לכל קטגוריה — כרטיס שמציג הכל ביחד:

```
🏗️ עבודות שלד

  ביצוע עבודה (מיומנים):
    בטון B30         ████████░░  75/120 מ"ק  62%
    ברזל זיון        ██████░░░░   6/8 טון    75%
    עבודת שלד        ███████░░░  32/45 יום   71%

  רכש:
    הוזמן 85%  |  סופק 60%

  קבלני משנה:
    קבלן שלד — שולם 180K מ-450K (40%)

  תקציב:
    תכנון: 520,000  |  ביצוע: 340,000  |  נוצל 65%

  ─────────────────────────────
  התקדמות כוללת: 68%  ← ממוצע משוקלל של כל המשימות
```

### חישוב התקדמות כוללת per קטגוריה

```
כל משימה בקטגוריה:
  - חומר: executedQty מהיומנים (אם יש) או % סופק מהרכש
  - עבודה: executedQty מהיומנים
  - קב"מ: % שולם מסה"כ
  - כולל: executedQty מהיומנים

ממוצע משוקלל (לפי ערך כספי) = אחוז התקדמות הקטגוריה
```

### שורת סיכום פרויקט

```
סה"כ פרויקט:
  התקדמות: 52%          ← ממוצע משוקלל כל הקטגוריות
  תקציב נוצל: 48%      ← הוצאות בפועל ÷ תכנון
  גבייה: 30%            ← נגבה ÷ ערך חוזה

  ⚠️ פער: התקדמות 52% אבל גבייה רק 30% — צריך לגבות!
```

## פיצ'ר 4: הוצאת הזמנת רכש (PDF)

### מה קיים
הזמנות כבר נשמרות ב-`purchase.orders[]`. חסר רק מסמך.

### שינויים

**Procurement.jsx** — כפתור "הפק הזמנה" ליד כל order:
```
handleExportPO(purchase, order) → window.open עם HTML
```

**מסמך הזמנה:**
```
┌─────────────────────────────────┐
│  [לוגו]         הזמנת רכש      │
│                 מס׳: PO-001    │
│                 תאריך: 01/05   │
├─────────────────────────────────┤
│  לכבוד: [ספק]                  │
│  פרויקט: [שם]                  │
│  כתובת: [כתובת]                │
├─────────────────────────────────┤
│  פריט │ יחידה │ כמות │ מחיר    │
│  בטון │ מ"ק   │ 45   │ 460₪   │
├─────────────────────────────────┤
│  סה"כ + מע"מ                   │
│  תאריך אספקה נדרש              │
│  הערות                         │
│  חתימה: ___________            │
└─────────────────────────────────┘
```

**אופציה מתקדמת — הזמנה מרוכזת לספק:**
- כפתור "הזמנה מרוכזת" → בוחרים ספק → כל הפריטים שלו → PDF אחד

## פיצ'ר 5: תוספות ושינויים (Change Orders)

### מודל נתונים
```
ChangeOrder = {
  id, projectId, number: "CO-001", date,
  status: "draft" | "sent" | "approved" | "rejected",
  description: "שינוי מיקום דלת + תוספת ארון",
  approvedBy: "", approvedDate: null,
  items: [{
    name, category, unit, quantity,
    type: "material" | "labor" | "subcontractor" | "combined",
    costPrice, clientPrice, description
  }],
  totalCost, totalSell
}
```

### Routes
| Route | קומפוננטה | סיידבר? |
|-------|----------|---------|
| `/project/:id/changes` | ChangeOrders | כן |
| `/change/:projectId/new` | ChangeOrderForm | כן |
| `/change/:projectId/:changeId` | ChangeOrderForm | כן |
| `/approve/:projectId/:changeId` | ChangeOrderApproval | לא |

### דפים
**ChangeOrders.jsx** — רשימה + סטטיסטיקות
**ChangeOrderForm.jsx** — יצירה/עריכה + כפתור "שלח ללקוח"
**ChangeOrderApproval.jsx** — דף חיצוני ממותג, לקוח רואה ומאשר

### חיבור לגבייה
- **רגיל:** אישור → אבן דרך חדשה "תוספות #X" עם סכום קבוע
- **BOQ:** אישור → סעיפים חדשים ב-boqQuote.items → מופיעים בחשבון חלקי

### חיבור ל-Layout, Dashboard, ProjectOverview
- סיידבר: לינק "תוספות ושינויים"
- דשבורד: התראה "X תוספות ממתינות לאישור"
- סקירה: ערך חוזה = מקורי + תוספות מאושרות

## פיצ'ר 6: חיבור ביצוע → גבייה

### לוגיקה

**אבני דרך (רגיל):**
כל אבן דרך מקושרת לקטגוריות. כשההתקדמות הכוללת של הקטגוריות המקושרות = 100%:
→ התראה: "אבן דרך [סיום שלד] מוכנה לגבייה"

**חשבון חלקי (BOQ):**
אחוזי הביצוע מהיומנים מציעים אוטומטית את הכמויות בחשבון החלקי הבא:
→ "בטון B30: בוצעו 15 מ"ק מאז החשבון האחרון — להכניס?"

### איפה
- ProjectAlerts — התראה "קטגוריה X הושלמה"
- BOQBilling — הצעת כמויות מיומנים
- ProjectTasks (כרטיס קטגוריה) — מראה מתי אפשר לגבות
